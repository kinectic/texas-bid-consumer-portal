# Bid Reliability Checklist

## Purpose
Define what "reliable enough" means before the janitorial bid monitoring service is treated as pilot-ready for paying clients.

## Reliability Standard
The service is not considered pilot-ready until it can repeatedly do the following:
- check the priority sources on a consistent cadence
- identify likely janitorial/custodial opportunities with low noise
- avoid obvious false positives most of the time
- package findings into clear usable briefs
- maintain a documented workflow that can be repeated without guesswork

## Phase 1 — Build
- [ ] Top source list defined
- [ ] Monitoring targets ranked
- [ ] Include / exclude keywords defined
- [ ] Brief template defined
- [ ] Prospect list structure defined

## Phase 2 — Test
- [ ] Re-check top 10–15 sources across multiple days
- [ ] Log each monitoring pass with date/time
- [ ] Record whether each source had relevant items, irrelevant items, or no changes
- [ ] Capture at least several real janitorial/custodial signals, whether open or recent historical examples
- [ ] Note common false positives (supplies, unrelated facilities work, generic services)

## Phase 3 — Tighten
- [ ] Refine include / exclude keywords based on false positives
- [ ] Refine source-priority ranking based on actual usefulness
- [ ] Improve brief format where information is routinely missing or unclear
- [ ] Separate strong sources from weak/noisy sources

## Phase 4 — Pilot Ready Criteria
Treat the service as pilot-ready only if most of these are true:
- [ ] Monitoring rhythm has been tested over multiple days
- [ ] Top sources are producing enough relevant signal to justify the service
- [ ] Sample pack contains credible real examples
- [ ] A small pilot promise can be stated honestly without overclaiming
- [ ] Internal workflow is stable enough to repeat

## Operating Rules
- Never claim full coverage.
- Never guarantee clients will not miss bids.
- Position the service as filtered monitoring and opportunity visibility, not certainty.
- Use a pilot framing until the workflow proves itself.

## Minimum Pilot Promise
Acceptable promise:
- We monitor a defined set of DFW public procurement sources for janitorial-related opportunities and send filtered summaries of relevant items we identify.

Unacceptable promise:
- We catch every bid.
- We guarantee no missed opportunities.
- We have complete coverage of all relevant public sources.

## Monitoring Log Fields
- Date checked
- Source name
- URL / portal
- Relevant janitorial signal found? (Y/N)
- Brief created? (Y/N)
- False positive notes
- Follow-up needed
