# Texas Contract MVP Mockup

## Home Page

### Headline
Texas Public Contract Intelligence

### Subheadline
A structured view of fragmented public procurement data — starting with a DFW facilities and janitorial wedge.

### Intro Copy
Most public contract information is scattered across county portals, school district bidding systems, airport procurement pages, transit board documents, and archived award records. This platform organizes that mess into a usable searchable system.

### Primary Filters
- Keyword
- Region
- Category
- Status
- Entity Type

### Summary Counters
- Entities tracked
- Sources tracked
- Opportunities indexed
- Awarded records
- Historical signal records

## Opportunity Feed Card
Each record card should show:
- Title
- Entity
- Geography
- Status badge
- Category
- Confidence level
- 1–2 sentence summary
- Source link

Example:
- **Janitorial Services for Passenger Facilities**
- DART
- DFW
- Awarded
- Janitorial / Transit Facilities Cleaning
- High confidence
- Board-document evidence of janitorial services tied to passenger facilities.
- [View Source]

## Entity Detail Page
Show:
- entity name
- entity type
- procurement systems used
- source links
- related records
- notes on why the entity matters

Example useful notes:
- uses IonWave
- strong custodial purchase history
- active school/public-facility footprint

## Source Detail Page
Show:
- source name
- parent entity
- procurement system type
- status (active / noisy / archive / weak)
- source URL
- related records

## Status Legend
- Open = currently active opportunity
- Awarded = completed or awarded procurement item
- Historical Signal = proof the entity buys in this category, but not currently confirmed open
- Weak Signal = insufficient confidence to present prominently

## MVP User Experience Goal
A user should quickly understand:
- which buyers matter
- which sources are real
- which records are open vs historical
- why this is more useful than checking portals manually
