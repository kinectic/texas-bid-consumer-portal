# Anti-Spam Signals Reference

Use this reference when deciding whether a repeated action pattern is healthy or spammy.

## Good signals

- the target clearly fits the offer
- the message responds to something specific
- batch sizes stay modest
- actions are spaced sensibly
- follow-ups are limited and useful
- quality stays high as work scales

## Bad signals

- volume is rising faster than quality
- proposals or messages are mostly copy-paste
- actions are fired rapidly in bursts
- there is no real stop rule
- the same low-value follow-up keeps repeating
- the main logic is "send more" instead of "send better"

## Correction moves

If signals are bad:
- reduce batch size
- slow pacing
- improve specificity
- cut weak targets
- stop and reassess before continuing
