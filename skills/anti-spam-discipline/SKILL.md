# Anti-Spam Discipline

Use this skill when repeated actions could drift into spammy, low-quality, or platform-risky behavior.

## Goal

Behave with restraint, selectivity, and relevance.

The purpose is not to bypass detection.
The purpose is to avoid acting like a spammer in the first place.

## Core Rule

Do not optimize for maximum volume.
Optimize for:
- relevance
- timing
- quality
- variation
- restraint
- expected value per action

## First Check

Before any repeated action, ask:
- is this relevant to the recipient or target?
- is this action necessary?
- is the content too repetitive?
- is the pacing too fast?
- would a reasonable human do this many this quickly?
- is this likely to annoy, clutter, or look low-effort?

If the answer looks bad, slow down or stop.

## Volume Rule

Do not blast large numbers of low-quality actions.

Default approach:
- start small
- assess response quality
- expand only if results justify it
- stop when returns are weak or signals look negative

## Pacing Rule

Avoid bursty behavior.

For repeated actions:
- batch modestly
- pause between batches when appropriate
- do not fire off long runs of nearly identical actions in a compressed window
- leave room to assess outcomes before sending more

## Relevance Rule

Each action should have a clear reason to exist.

Good reasons:
- the target fits the offer
- the proposal addresses the actual request
- the follow-up adds value
- the timing makes sense

Bad reasons:
- filling quota
- spraying broadly just to see what sticks
- repeating the same generic pitch everywhere

## Quality Rule

Prefer fewer higher-quality actions over many weak ones.

Each external action should try to be:
- specific
- credible
- short enough to respect attention
- tailored where it matters
- honest about capabilities

## Repetition Rule

Do not reuse the exact same message shape too many times in a row.

Vary when appropriate:
- opening line
- angle
- examples used
- framing of the value

But do not fake personalization.
Only vary where the change is real and relevant.

## Follow-Up Rule

Follow up only when there is a real reason.

Good follow-up:
- short
- respectful
- infrequent
- adds clarity or value

Bad follow-up:
- repeated nudges with nothing new
- pressure tactics
- too many touches too quickly

## Marketplace Rule

For places like Upwork or similar platforms:
- do not mass-apply blindly
- target jobs that are plausible fits
- prefer smaller numbers of stronger proposals
- avoid generic copy-paste proposals
- avoid applying so broadly that quality collapses

## Stop Signals

Pause or stop when:
- response rate is very poor
- no positive signal appears after a reasonable sample
- quality is dropping because speed is too high
- the work starts looking repetitive and generic
- the platform or workflow feels like it is rewarding spammy behavior
- continuing would mostly add volume without value

## Human-Like Standard

Use this practical test:

> Would this look like thoughtful selective work from a real person, or like a machine pushing volume?

If it looks like volume pushing, redesign it.

## Safety Rule

Never use this skill to impersonate a person deceptively or to evade platform safeguards.

Use it to reduce spamminess, not to disguise abuse.

## Output Format

When planning repeated outward actions, summarize briefly:
- target type
- batch size
- pacing
- stop signal
- what makes the action relevant

## Guiding Principle

Use this rule:

> The best way not to look spammy is not to do spammy things.
