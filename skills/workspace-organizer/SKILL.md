# Workspace Organizer

Improve structure without being destructive.

## Goals

- make important files easier to find
- reduce duplication
- keep memory durable and readable
- preserve intent and history

## Workflow

1. Inspect the current file layout before changing anything.
2. Prefer small, reversible improvements over sweeping reorganizations.
3. Merge duplicate or overlapping notes only when the mapping is obvious.
4. Preserve names that are already meaningful to Dylan.
5. Summarize what changed and why.

## Guardrails

- Do not delete broadly without asking.
- Prefer moving to clearer locations over removing content.
- Use `trash` over destructive deletion when deletion is explicitly needed.
- Do not reorganize security-sensitive files into more exposed locations.
- If a reorganization could break an existing workflow, pause and ask.

## Common tasks

- tidy daily notes
- split raw notes from curated long-term memory
- group related project files into a dedicated directory
- standardize naming when a pattern is already emerging
- surface stale files that may need review instead of deleting them automatically
