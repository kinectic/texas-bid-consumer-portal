# Weekly Bands Reference

Use this reference when Codex weekly usage needs a concrete operating posture.

## Bands

- Green: 70% to 100% remaining
- Yellow: 40% to 69% remaining
- Orange: 20% to 39% remaining
- Red: below 20% remaining

## Meaning

### Green
- normal disciplined work
- avoid obvious waste

### Yellow
- conserve moderately
- batch more
- cut unnecessary churn

### Orange
- prioritize higher-value work
- avoid long exploratory loops
- prefer one-pass execution

### Red
- essential work only
- emergency conservation until reset

## Checkpoints

Use `openclaw status --usage`:
- before a significant work block
- after a heavy session
- when Dylan asks about usage
- when behavior should shift by band
