# Texas Contract MVP Website

## Purpose
Define the minimum viable website view for the future Texas contract intelligence platform.

## MVP Goal
Show useful, trustworthy, structured procurement intelligence from the DFW wedge before trying to cover full Texas breadth.

## MVP Should Display

### 1. Opportunity Feed
A list of structured records with:
- title
- entity
- geography
- category
- status
- confidence level
- short summary
- source link

### 2. Filters
Minimum filters:
- geography
- category
- status
- entity type
- keyword search

### 3. Status Types
The website must visibly distinguish:
- **open**
- **awarded**
- **historical signal**

This is critical.
If these are mixed together without clear labels, trust drops immediately.

### 4. Record Detail View
Each record should show:
- normalized title
- buying entity
- source type
- supporting excerpt
- link to source
- notes about why it is relevant
- confidence level

### 5. Source / Entity Pages
Simple profile-style pages for entities and sources showing:
- who the buyer is
- procurement system used
- what categories they appear to buy in
- why they matter

## MVP Home Page Positioning
The homepage should explain:
- this is procurement intelligence, not just a raw list of bids
- the platform consolidates fragmented Texas public procurement signals
- the early wedge is DFW facilities/janitorial-related procurement intelligence

## What The MVP Should Not Try To Do Yet
- full Texas coverage
- every category
- user accounts
- billing
- complex saved-search logic
- overbuilt dashboards
- claims of complete coverage

## MVP Success Test
The MVP is good enough if a user can:
- quickly understand what kinds of records are here
- distinguish open vs historical vs awarded
- find relevant DFW records fast
- see why the platform adds value beyond a raw procurement portal
